OwinFramework analysis reporter middleware configuration

The analysis reporter middleware in the OwinFramework support the following configuration options:

Property Default Current Description
Path {path.default} {path}

Configures the relative path within your application where the analysis reporter will be available.

For example if your web site is hosted at http://mydomain.com/ and you set this path property to /owin/analysis then the full url of the visualizer will be http://mydomain.com/owin/analysis

Enabled {enabled.default} {enabled}

The Enabled property allows you to turn off the analysis reporter.

You might want to disable the Owin analysis reporter in a production web site to prevent public users from seeing internal details of your application implementation. You could also limit the availability of the analysis reporter by placing middleware in front of it that restricts access to only certain people instead of disabling it.

RequiredPermission {requiredPermission.default} {requiredPermission}

The RequiredPermission property allows you to restrict acess to the analysis reporter.

For this to work you must have some Authorization middleware configured in your Owin pipeline. The name of the permission that you specify here is any permission recognized by the Authorization middleware that you configured.

DefaultFormat {defaultFormat.default} {defaultFormat}

The DefaultFormat property specifies the format of the report when the request does not contain an Accept header. This must be a valid and supported MIME type.

For this to work you must have some Authorization middleware configured in your Owin pipeline. The name of the permission that you specify here is any permission recognized by the Authorization middleware that you configured.

This middleware is configured the same way as all other middleware in the Owin Framework; you need to construct a class that implements IConfiguration then add a fluid .ConfigureWith() when you register this middleware with the Owin Framework builder. Please read Owin Framework documentation on configuration for full details with examples.
